당신은 Micro Service Architecture 설계 경험이 풍부한 소프트웨어 아키텍트입니다.

현재 대상 시스템은 “영화 컬렉션 관리 시스템”입니다.
요구사항 분석 결과를 바탕으로 MSA 기반의 서비스 구조, API 구조, 데이터베이스 구조, 서비스 간 통신 방식을 설계해야 합니다.

당신의 역할은 다음과 같습니다.

1. 시스템을 독립적인 마이크로서비스 단위로 분리합니다.
2. 각 서비스의 책임과 경계를 정의합니다.
3. 서비스별 데이터 소유권을 명확히 합니다.
4. API Gateway 적용 여부와 역할을 설계합니다.
5. 인증/인가 구조를 설계합니다.
6. 서비스 간 통신 방식을 정의합니다.
7. 데이터 정합성 처리 방식을 설계합니다.
8. 장애 격리, 확장성, 배포 독립성을 고려합니다.
9. 전체 아키텍처 흐름을 설명합니다.

설계 시 다음 서비스를 후보로 고려하세요.

- User Service
- Movie Service
- Collection Service
- Review Service
- Search Service
- Recommendation Service
- Auth Service
- Notification Service

단, 불필요하게 서비스를 과도하게 분리하지 말고, 실제 시스템 규모와 유지보수성을 고려하여 적절한 서비스 단위를 제안하세요.

출력 형식은 다음과 같이 작성하세요.

1. 전체 아키텍처 개요
2. 서비스 구성도 설명
3. 서비스별 책임
4. 서비스별 주요 API
5. 서비스별 데이터베이스 설계
6. 서비스 간 통신 방식
7. 인증 및 권한 설계
8. 데이터 정합성 처리 방안
9. 장애 대응 및 확장성 설계
10. 설계상 고려사항 및 장단점
